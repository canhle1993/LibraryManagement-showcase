# Challenges And Solutions

## Challenge 1: Showing a Private Project Publicly

The actual implementation repository is private because I do not want to expose the full codebase publicly.

### Solution

I created this separate showcase repository that contains:

- architecture and flow explanations
- screenshots
- feature summaries
- role and contribution notes
- database diagram

This makes the project reviewable without exposing source code.

## Challenge 2: Making A Desktop App Easy To Review

Desktop applications are harder to evaluate quickly than web projects because there is no public URL.

### Solution

I organized the public material around:

- concise README sections
- screenshots for major modules
- system-flow documentation
- tech stack and integration notes

## Challenge 3: Building More Than Basic CRUD

A library app can easily become a simple form-based project.

### Solution

We expanded the scope with:

- QR-assisted inventory workflows
- Vbee text-to-speech integration
- damage status handling
- recycle bin recovery
- reporting and historical tracking

These features made the project more realistic and more portfolio-worthy.
